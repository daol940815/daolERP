운영 오픈과 법인 복제 배포 시 이 순서를 따른다. (기획서 12장 미확정 사항 중 운영 확정이 필요한 항목 포함)

| # | 항목 | 상태 |

## 2. 인프라 준비

- [ ] 서버 준비 (Docker + Docker Compose)
- [ ] `.env.production.example` → `.env` 복사, `POSTGRES_PASSWORD`·`JWT_SECRET` 강한 값으로 설정
- [ ] `docker compose -f docker-compose.prod.yml up -d --build`
  - ⚠️ Dockerfile 은 개발 환경에서 docker 데몬 부재로 빌드 미검증 — **최초 배포 시 빌드 확인 필요**
- [ ] DB 백업 스케줄 구성 (pg_dump 일 1회 이상 — 근태 기록 3년 보존 의무)

## 3. 초기 설정 (관리자 로그인 후)

- [ ] `admin@daolerp.local` 로그인 → **즉시 비밀번호 변경** (`POST /api/auth/change-password`)
- [ ] 시스템 설정: 회사명/사업자번호/대표자/주소, SMTP(이메일 알림 사용 시)
- [ ] 정책 관리: 실제 근무정책 등록·전사 기본값 지정, 연차정책 확인, 휴가유형 점검
- [ ] 휴일: 당해 연도 공휴일 Import (Excel) 또는 수동 등록
- [ ] 부서 등록 + **부서장 지정** (지정 없으면 승인라인 DEPT_HEAD 해석 불가)

## 4. 직원/계정 등록

- [ ] 직원 Excel Import (dryRun 검증 → 확정) — 열: 사번|이름|입사일|부서명|직급코드|고용형태코드
- [ ] 계정 발급 (사용자 관리) — 초기 비밀번호는 개별 전달, 최초 로그인 후 변경 안내
- [ ] 역할 배정: 부서장 → APPROVER, 인사팀 → HR

## 5. 데이터 초기화

- [ ] **연차 초기값 Import** — 오픈 시점 잔여 연차 (열: 사번|잔여일수|발생일|사용기한)
  - 기존에 회계연도 기준으로 관리했다면: 잔여를 그대로 초기값으로 넣고, 이후 발생은 입사일 기준 자동 (기획서 결정 #1)
  - 자동 발생 배치는 **입사일 기준 전체 이력**을 생성하므로, 초기값 Import 와 병행 시 이중 발생 주의 — 둘 중 하나만 사용:
    - (권장) 신규 오픈: `leave-grant` 배치 활성화 → 법정 기준 자동 발생 + 필요 시 수동 조정
    - 이관 오픈: 초기값 Import 만 사용, 배치는 오픈 다음 달부터 활성화
- [ ] 근무일정 생성: 당월 + 익월 (근무일정 관리 화면 또는 배치 수동 실행)
- [ ] 스케줄러 작업 활성화: work-schedule-generate, leave-grant/expire/promotion-alert, weekly-hours-check, notification-dispatch (스케줄러 화면에서 is_active — 현재 기본 비활성)

## 6. 오픈 검증 (리허설 — 통합 테스트가 자동 수행하는 항목)

`apps/api` 에서 `pnpm test:integration` — 신규 DB 에서 아래를 자동 검증 (28항목):

- 연차 자동 발생이 독립 계산과 **전수 일치** (기획서 10.4 GO/NO-GO 기준)
- 엔진 판정 표본 대조 (주중 결근/주말 휴무/보정 반영/휴가)
- 보정·휴가·초과근무 신청→승인→반영 전 과정
- 월 마감 → 스냅샷 → 마감 가드 → 리포트 SNAPSHOT/LIVE 전환
- Excel Import 2단계, 비밀번호 변경, 접근 제어

## 7. 운영 중 정기 확인

- [ ] 관리자 대시보드 일 1회 (미출근·승인 대기·52시간 임박)
- [ ] 월초: 전월 마감 실행 → 급여용 Excel 내보내기
- [ ] 스케줄러 실행 이력 주 1회 점검 (실패 작업)
- [ ] 첨부파일 보존 기간 정책 확정 후 파기 배치 등록 (기획서 4.8 — 미구현, 후속)

## 8. 병합 GO/NO-GO 판단 기준 (기획서 10.4)

운영 검증 1~3개월 후:
- [ ] 1회 이상 월 마감 실데이터 완료 (검증→마감→스냅샷→내보내기)
- [ ] 연차 자동 발생 = 수기 계산 전수 일치
- [ ] 엔진 판정 = 실제 근태 표본 대조 일치
- [ ] 사용자 불편 사항 처리 완료
