# 서버 준비 가이드 — 서버가 없는 상태에서 시작하기

근태 시스템은 직원 수십 명 규모 기준 **2 vCPU / RAM 4GB / 디스크 50GB** 서버 1대면 충분하다.
아래 두 경로 중 하나를 선택한다. (오픈 절차 전체는 `오픈_체크리스트.md` 참고)

---

## 경로 A. 클라우드 가상서버 (권장 — 월 1~3만 원)

재택·외근·모바일 출퇴근 체크가 필요하거나, 정식 운영이 목표라면 이 경로.
법인 복제 운영(기획서 1.3 — 법인당 서버 1대) 구조와도 일치한다.

### A-1. 서버 구매 (약 10~20분)

아무 업체나 가능. 개인정보(근태 기록) 특성상 **국내(서울) 리전 권장**.

| 업체 | 상품 예 | 참고 |
|---|---|---|
| 네이버 클라우드 | Micro/Compact 서버 | 국내, 세금계산서 발행 용이 |
| 가비아 g클라우드 | 2vCPU/4GB | 국내, 도메인과 함께 관리 가능 |
| AWS Lightsail | $12/월 플랜 (2GB면 $10) | 서울 리전 선택, 관리 화면 단순 |
| Vultr / DigitalOcean | $12~18/월 | 해외 업체, 서울 리전 있음 |

구매 시 선택할 것:
- OS: **Ubuntu 24.04 LTS** (또는 22.04)
- 공인 IP 1개 (기본 포함되는지 확인)
- SSH 접속 키 또는 비밀번호 설정

### A-2. 도메인 (외부 접속 시 — 연 1~2만 원)

- 보유 도메인이 있으면 서브도메인 사용: 예 `gt.회사도메인.com`
- 없으면 가비아 등에서 구매 후, DNS 관리에서 **A 레코드**를 서버 공인 IP로 지정

### A-3. 서버 설정 + 배포 (약 20분)

```bash
# 1) 서버 SSH 접속 후 초기 설정 (Docker, 방화벽, 스왑, 클론까지 자동)
curl -fsSL https://raw.githubusercontent.com/daol940815/daolERP/main/deploy/setup-server.sh | bash
# (비공개 저장소면 스크립트 안내에 따라 토큰 포함 URL 로 클론)

# 2) 환경 설정
cd ~/daolERP
cp .env.production.example .env
openssl rand -hex 48       # 출력값을 JWT_SECRET 에 붙여넣기
nano .env                  # POSTGRES_PASSWORD, JWT_SECRET, DOMAIN, (HTTPS 시) WEB_PORT=127.0.0.1:8080

# 3) 기동 — 도메인+HTTPS (외부 접속)
docker compose -f docker-compose.prod.yml -f deploy/docker-compose.https.yml up -d --build
#    또는 사내 전용 HTTP
docker compose -f docker-compose.prod.yml up -d --build

# 4) 초기 데이터 시드 (역할/권한/기본 정책/관리자 계정 — 최초 1회, 멱등이라 재실행 안전)
docker compose -f docker-compose.prod.yml exec api npx ts-node prisma/seed.ts
```

> ⚠️ Dockerfile 은 개발 환경(도커 데몬 부재)에서 실빌드 검증을 못 했다.
> 최초 `--build` 에서 오류가 나면 로그를 개발 담당(Claude)에게 전달할 것.

### A-4. 확인

- 브라우저에서 `https://도메인` (또는 `http://서버IP`) 접속 → 로그인 화면
- `admin@daolerp.local / admin1234!` 로그인 → **즉시 비밀번호 변경**
- 이후 `오픈_체크리스트.md` 3단계(초기 설정)부터 진행

### A-5. 백업 (필수 — 근태 기록 3년 보존 의무)

```bash
# 매일 03시 DB 덤프 (서버 crontab -e 에 추가)
0 3 * * * docker compose -f ~/daolERP/docker-compose.prod.yml exec -T postgres pg_dump -U daolerp daolerp | gzip > ~/backup/daolerp-$(date +\%F).sql.gz
```
백업 파일은 주기적으로 서버 밖(구글 드라이브, NAS 등)으로 복사한다.

## 경로 B. 사무실 PC (비용 0원 — 파일럿용)

사무실 안에서만 쓰는 파일럿이라면 남는 PC(또는 10만 원대 미니PC)로 시작 가능.

1. PC에 Ubuntu 설치 (또는 Windows + Docker Desktop — 상시 켜둘 수 있어야 함)
2. 위 A-3 과 동일하게 설정하되 HTTP 모드로 기동 (`docker-compose.prod.yml` 만)
3. 직원들은 사내망에서 `http://PC내부IP` 로 접속
4. 제약을 인지할 것: PC 꺼짐/정전 = 출퇴근 체크 불가, 외부 접속 불가(재택·모바일 미지원), 백업 직접 관리
5. 파일럿 통과 후 정식 오픈 시 경로 A 로 이전 — DB 덤프 복원이면 되므로 이전 비용은 작다

## 자주 묻는 것

- **사양이 더 필요해지면?** 직원 100명+ 이 되어도 4GB 로 충분한 구조(계산은 조회 시점, 대용량 배치 없음). 느려지면 클라우드 콘솔에서 상위 플랜으로 리사이즈.
- **법인 추가(올푸름 등)는?** 법인마다 위 절차 반복 — 같은 저장소, 별도 서버 + 별도 `.env` + 시스템 설정만 변경 (기획서 1.3).
- **도메인 없이 HTTPS 는?** 불가(인증서는 도메인 기준). 외부 접속이 필요하면 도메인부터 준비.
