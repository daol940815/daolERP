# daolERP 전체 계획서 (2026-07-07)

> 목표: **"처리하면 회계가 자동 생성"** — 업무 데이터(ERP·통장·카드·세금계산서)를 올리고 확정하면
> 분개·원장·손익이 자동으로 만들어지는 시스템.
> 검증 기준: **2025년 데이터로 출력한 원장이 더존 총괄원장 금액과 동일하거나 근사(가능한 0)**.

운영 원칙 (전 단계 공통)
- 자동 분류는 **추천까지만**, 확정은 항상 사용자가 한다. 신뢰도 수치는 내부용(화면엔 자동추천/검토필요/선택요청 3단계만).
- 데이터 자동 병합 금지, 병합·확정에는 이력을 남긴다.
- 진행 전 보고 → 승인 → 실행 → **실데이터 검증 보고서** 사이클 유지.
- 분개는 전부 멱등(post_journal RPC) — 재확정·해제 시 장부가 자동 정정된다.

## 1단계 — 분류 도구 (시스템 작업, 즉시 착수 가능)

### 1-A. 매입 세금계산서 일괄 분류 도구
- **거래처 기본계정** `vendors.default_account_id` 신설 (마이그레이션 1건, 사장님 적용)
- 추천 파이프라인: ① 거래처 확정 이력 과반 → ② 거래처 기본계정 → ③ 품목명 키워드 → 없으면 선택요청
- **거래처별 그룹 화면**: "늘푸른팜 — 미분류 24건 · 8.4M · 추천 상품매입(이력 31건)" → 체크 → 일괄 확정 → 분개 자동
- 확정할수록 학습이 쌓여 다음 달부터는 대부분 자동추천 상태로 유입

### 1-B. 통장 거래 분류 도구
- 유형 자동 판별 후 ⑤만 분류 대상으로:
  ① 내부이체(제외·대체분개) ② 카드사 정산입금(기존 흐름) ③ 세계 매칭 입출금(채권·채무 상계)
  ④ 카드대금 출금(기존 상계) ⑤ **나머지: 수수료·이자·급여·세금 등 → 이 도구로 확정**
- 추천: 거래처 이력 → 기본계정(1-A와 공유) → 적요 키워드
- **유사거래 묶음 일괄 확정** (같은 적요 패턴 N건 한 번에)
- 안전장치: ①~④ 유형에 비용 계정 확정 시 경고(이중계상 1차 방어)
- ③ 유형의 상계 분개 규칙은 구현 전 분개 시안 보고 → 승인 후 적용 (3단계와 직결)

**산출물**: 마이그레이션, 그룹 분류 화면 2종, 실데이터 검증 보고서

## 3단계 — 월별 손익 정확화 (작업목록 6·7번)

- **[결정 필요] 매출원가 방식(6번)**: ERP 원가 기준 vs 매입(상품매입 5002) 기준.
  상품매입 계정과 ERP 원가의 관계 정리 — 이중계상 방지가 핵심.
- **손익 분개 기반 전환(7번)**: 월별 손익현황을 분개(원장) 집계로 전환.

## 진행 순서 요약

```
1단계(분류 도구) → 2단계(데이터 확정, 병행) → 3단계(손익 전환) → 4단계(2025 더존 대사)
```

각 단계는 "보고 → 승인 → 실행 → 실데이터 검증 보고서" 사이클로 진행한다.
