# 매입 사이클 실데이터 검증 보고서

작성일: 2026-07-09 · 대상 기간: 2025-01 ~ 2026-07 · 데이터: 운영 DB (거래처×월 셀 1,732개, 매입처 325곳)

설계(docs/purchase-cycle-design.md v3) 로드맵 6단계 중 5단계(재검토·확인 이력)와 3단계(거래처 진행상태 화면)를
구현한 뒤, 운영 데이터 전체로 상태 엔진을 검증한 결과입니다.

---

## 1. 정합성 검증 — 통과

| 검증 항목 | 방법 | 결과 |
|---|---|---|
| 거래처 진행상태 누계 = 월별 셀 합 | (주)웰크리 18개월 | ERP·계산서·지급 3축 모두 일치 |
| 월 드릴다운 근거 레코드 합 = 집계 셀 | 웰크리 4개월 + 파인갤러리 3개월 | 7개월 전부 원단위 일치 (ERP 품목·계산서·지급) |
| 지급 2원천(결제연결·2001 확정출금) 중복 방지 | 파인갤러리 지급월 | 이중집계 없음 |
| 확인 기록(061) 미적용 시 동작 | 운영 DB (061 미적용 상태) | 예외 화면은 정상 동작, 확인 버튼만 비활성 + 명확한 안내 |

## 2. 검증 중 발견·수정한 버그 2건

### ① RPC 결과 1,000행 절단 — 거래처 42% 누락 (치명)
- 셀이 총 1,732개인데 API가 PostgREST 기본 한도(1,000행)만 받아 **뒤쪽 거래처 732셀이 통째로 빠진 채**
  예외를 계산하고 있었음 (지급 대기 195곳으로 표시 — 실제는 309곳).
- 수정: 예외 API·거래처 API 모두 RPC 호출을 페이지네이션(fetchAllRows). 수정 후 1단계 검증 때의
  전수 숫자(완료 352 · 경비성 1,169)와 정확히 일치함을 확인.

### ② 지급 비교 기준 착오 — 완납 거래처가 "과다 지급"으로 (부가세)
- 지급은 부가세 포함 총액으로 이뤄지는데 계산서 축은 공급가로 비교 → 완납 거래처 13곳이
  **초과율 정확히 +10%**로 과다 지급에 잡혔고, 미지급액은 부가세만큼 과소 계산됨.
- 수정: 062 마이그레이션으로 RPC에 계산서 총액(invoice_total) 추가.
  ERP↔계산서는 공급가(변경 없음), 계산서↔지급은 총액 기준으로 분리.
  면세 계산서(세액 0)가 있어 ×1.1 가정 대신 실데이터 합계 사용.
- 운영 데이터 시뮬레이션: 과다 지급 **13곳 → 0곳** (전부 부가세 착시 확인),
  미지급 합계 47.0억 → **50.7억** (부가세 반영, 정확한 값).

## 3. 예외 현황 (수정 반영, 2025-01 ~ 2026-07)

| 상태 | 곳/셀 | 비고 |
|---|---|---|
| 완료 | 352셀 | ERP ≈ 계산서 (공급가 ±10%) |
| 경비성 | 1,169셀 | ERP 무관 매입 — 정상 유형 |
| 지급 대기 | 309곳 | 미지급 합계 50.7억 (총액 기준) |
| 금액 차이 | 165셀 | ERP↔계산서 ±10% 초과 |
| 계산서 대기 | 19셀 | ERP 매입 있음·계산서 없음, 합계 46.9M |
| 과다 지급 | 0곳 | 062 반영 후 전부 해소 |

확인 필요(최우선) 466행 = 지급 대기 293 + 금액 차이 145 + 계산서 대기 15 (+과다 지급 13은 해소).

## 4. 원인 규명

### 지급 대기 309곳 — 대부분 "실제 미지급"이 아니라 "매칭 미진행"
- 270곳(87%)은 지급 기록이 전무. 회사가 대금을 안 준 것이 아니라
  **통장 출금에 거래처 태깅·결제연결이 아직 안 된 것**이 대부분 (통장 태깅 완료는 48곳뿐).
- 조치 경로(화면에 이미 구현): ① [1:1 정확 매칭 일괄 실행] → ② 행별 [지급 후보] 확정
  → ③ 남는 건 통장 화면에서 거래처 태깅 후 재시도.

### 금액 차이 165셀 — 원인 추정 분포
| 추정 원인 | 셀 | 의미 |
|---|---|---|
| 부분 발행 또는 미발행 | 91 | 계산서 < ERP — 거래처에 발행 요청 필요 가능 |
| ERP 미입력 또는 경비성 혼재 | 64 | 계산서 > ERP — ERP 주문 입력 누락 또는 경비 섞임 |
| 전월/익월 발행 (시차) | 10 | 인접월 합산 시 근사 — 정상 시차로 자동 강등 |

상위 사례: 다온글로비스 2026-05 (+29.7M, 부분 발행 추정) · 웰크리 2025-08 (+26.2M) ·
웰크리 2025-09 (-23.5M, ERP 미입력 추정) — 거래처 진행상태 화면에서 월 클릭으로 근거 레코드 확인 가능.

### 계산서 대기 19셀 (46.9M)
확인 필요 15셀: 잇츠프레쉬 2026-01 (6.7M) · 서우글로벌 2025-04~06 (6.1M) · 도기야 2025-08 (4.3M) ·
대현상회 2026-01 (3.2M) 등 — 세금계산서 수취 여부 확인 대상.
정상 대기 4셀은 2026-06 최근분 (발행 주기 내).

## 5. 남은 조치 (권고)

1. 마이그레이션 061(확인 이력)·062(계산서 총액) 적용 — 적용 전에도 화면은 동작하나
   확인 버튼 비활성 + 과다 지급 13곳 착시가 남음.
2. [1:1 정확 매칭 일괄 실행] 후 지급 대기 감소 폭 확인 → 지급 후보 확정 진행.
3. 계산서 대기 15셀(확인 필요)은 거래처에 발행 여부 확인.
4. 허용범위(±10%)·경과 기준은 운영하며 조정 (설계 §2-2).
