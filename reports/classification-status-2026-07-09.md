## 2. 축별 상세

### 매출 세금계산서 — 분류 완료
- 2,777건 중 2,761건(99.4%) 계정 확정, 분개 자동 생성 완료. 미분류 16건.
- 입금확인이 8건뿐인 것은 문제가 아니라 구조 차이: 매출 대금은 카드사·PG 정산으로
  들어와서 계산서 1장 = 입금 1건 대응이 원래 성립하지 않음. 매출 검증은 ERP·카드매출
  정산 쪽에서 하고 있음.

### 매입 세금계산서 — 분류 93%, 결제매칭이 이번 주 작업
- 계정 미분류 175건 (2025년 49건 / 2026년 126건) — 일괄 분류 화면에서 처리 가능.
- 결제확인(통장 출금과 연결) 297건. 오늘 수정한 1:1 자동매칭을 실행하면 약 546건이
  추가 연결될 예정 (금액·거래처가 정확히 일치하는 것만).
- 매입 사이클 현황(2025-01~2026-07, 거래처×월): 완료 352 · 경비성 1,169(정상) ·
  지급 대기 309곳 · 금액 차이 165 · 계산서 대기 19.
  지급 대기의 87%는 실제 미지급이 아니라 통장 연결 작업이 아직 안 된 것.

### 통장 거래내역 — 다음 최우선 작업 대상
- 7,301건 중 계정 확정 87건(1.2%). 추천이 붙어 있는 1,209건은 확인 후 확정만 하면 됨.
- 이체쌍(자사 계좌 간 이동) 835건 연결, 거래처 태깅 1,548건.
- 통장 분류 화면에 유형별 묶음 처리(이체/카드사 정산입금/카드대금/일반)가 준비되어
  있어 묶음 단위로 빠르게 진행 가능. 2025년분(5,106건)이 대부분.

### 법인카드 — 추천 완비, 확정만 남음
- 885건(2026년) 중 확정 43건. 654건은 2025년 세무사 분류를 학습한 추천이 붙어 있어
  확인 후 일괄 확정 가능. 추천 없는 188건만 개별 판단 필요.

### ERP 주문 — 연결 거의 완료
- 취소 제외 21,277품목 중 19,936건(93.7%) 매입처 연결. 미연결 1,341건은 별칭 미등록
  거래처 — 매입 사이클 예외 화면에서 거래처 단위로 정리하면서 자연스럽게 줄어듦.

## 4. 사장님 보고용 요약 (간단 버전)

- 세금계산서(매출·매입)와 카드매출은 계정 분류가 93~99% 끝났고, 장부(분개)도 자동으로
  만들어지고 있습니다. 회계 숫자의 뼈대는 완성 단계입니다.
- 지금은 "계산서와 실제 통장 출금을 연결하는" 매칭 작업 단계입니다. 자동으로 확실한
  것만 골라 연결하는 기능이 준비되어 있고, 애매한 것은 사람이 확인 후 확정합니다.
- 남은 큰 작업은 통장 거래내역 분류(약 7,300건)와 법인카드 확정(약 700건)인데,
  둘 다 시스템이 추천을 만들어 두어 확인·확정만 하면 되는 상태입니다.
