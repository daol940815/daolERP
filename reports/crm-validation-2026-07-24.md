# CRM(매출처 고객관리) 이행 사전검증 보고서

작성일: 2026-07-24 · 검증 환경: 로컬 PostgreSQL 16 (마이그레이션 001~070 전체 적용)
설계: `docs/customer-management-design.md` · 대상: 마이그레이션 069·070 + 이행 스크립트

## 요약

| 검증 | 결과 |
|---|---|
| 마이그레이션 001~070 순차 적용 | **전부 성공** (기존 68개 + 신규 2개, 오류 0) |
| 주문 ↔ 고객 매칭 (7,704 가상주문) | **100% 귀속** (미매칭 0) |
| 2단: DB 등급 계산 전수 대조 (3,301명) | **불일치 0** (합계·매출등급·연속성·거래플래그) |
| 시즌 백필 정확도 (7,704건) | 98.5% 일치 · **오분류 0** · 경계 과태깅 116 · 누락 2 |
| 신규/이탈·스냅샷 RPC | 정상 동작 (2026-07 스냅샷 3,301행 생성) |

검증 방법: 엑셀 원장(2025~26) 22,517라인을 가상 주문으로 로컬 DB에 적재(`sim_load_orders.py`),
2024년 9,651라인은 legacy 이관 경로로 적재한 뒤, 파이썬 독립 계산과 DB RPC를 고객 단위 전수 대조.

## 1단 — 엑셀 규칙 재현 vs 통계 시트 자체 값

통계 시트 3,011행 중 **2,973행(98.7%) 일치**. 불일치 38건(합계)·9건(플래그)·12건(등급)의
원인은 전부 **엑셀 캐시 스테일**로 확인:

> 사례 — (주)이안종합관리: 담당자 표기가 '이현숙 대표님'(2024) → '이현숙 님'(2025~)으로
> 데이터에서 수정됐으나 통계 시트 수식이 재계산되지 않아, 캐시값(295.6만)에는 옛 표기
> 시절 집계가 남아 있고 현재 데이터 기준 SUMIFS는 102.3만이 맞다.

즉 **현재 엑셀 파일의 통계 숫자 일부는 자기 데이터와도 안 맞는 상태**이며,
이는 설계에서 지적한 문자열 키·수동 재계산의 한계가 실제로 발생한 것. 시스템은 이 문제가
구조적으로 없다(매칭 키 학습 + 실시간 집계).

## 2단 — DB 구현 정확성 (핵심)

확정 규칙(취소·VIP·선결제·샘플·배송비류 제외)으로 파이썬이 독립 계산한 고객별
누적매출·매출등급·연속성등급·연도별 거래플래그를 DB RPC(`crm_contact_stats`)와
**3,301명 전수 대조 → 불일치 0건.**

과정에서 잡은 구현 결함 2건 (수정 완료):
1. 2024-12월에 주문된 '25설' 명절 건이 legacy 이관에서 누락 (예: 교보생명 3,800만원)
   → 2024년 날짜 라인은 구분이 어느 명절이든 legacy로 받도록 수정
2. 시즌 백필 SQL이 지점명 빈 고객을 NULL로 만들어 조인 실패(430건 누락)
   → 빈 문자열 유지로 수정 (누락 430→2)

## 3단 — 규칙 변경(VIP·샘플 제외) 영향

동일 모집단 3,241명 기준 분포 차이는 **±2명 수준** — 사실상 영향 없음
(엑셀도 구분 필터로 대부분의 VIP·샘플을 이미 제외하고 있었음).

|---|---|---|---|---|
| 엑셀 규칙 | 186 | 254 | 1,118 | 1,683 |
| 확정 규칙 | 185 | 255 | 1,119 | 1,682 |

## 발견 사항 (엑셀 대비 시스템의 개선점)

1. **엑셀 집계표의 신규/이탈(228/120, 102/217)은 필터가 걸린 화면의 SUBTOTAL 값**이었다.
   전체 모집단 기준 실제 값: 2025 신규 1,097 / 이탈 1,019 · 2026 신규 497 / 이탈 1,097.
   시스템의 2026 신규는 엑셀 키 한정 시 **497로 정확 일치**.
2. **통계 시트에 없던 거래 고객 290명 회수** — 2024 원장에만 111명, 2025~26 원장에만 179명
   (담당자 표기 변경으로 키가 어긋난 경우 포함). 전부 자동 생성(메모로 출처 표시)했고,
   동일인 여부는 화면에서 병합으로 확정한다(자동 병합 금지 원칙).
3. 시즌 백필 과태깅 116건(1.5%)은 같은 날 같은 고객이 명절+상시 주문을 함께 낸 경계 사례 —
   주문번호가 없는 엑셀의 구조적 한계로, 화면에서 개별 수정 가능(오분류는 0건).
