# 💾 daolERP 백업 · 복구 전략

회사 ERP 데이터(매출·매입·통장·카드·영업이력)는 손실 시 복구가 어렵습니다.
아래 3중 백업 전략을 권장합니다. **지금 당장은 1·2단계만으로 충분**하며,
규모가 커지면 3단계(자동화)를 도입하세요.

---

## 한눈에 보기

| 단계 | 방법 | 주기 | 난이도 | 비고 |
|------|------|------|--------|------|
| 1 | **Neon 자동 백업(PITR)** | 자동(상시) | ★ | 가입만 하면 기본 제공 |
| 2 | **수동 덤프(`pg_dump`)** 파일 보관 | 주 1회 권장 | ★★ | 회사 PC/클라우드에 저장 |
| 3 | **자동 백업 스케줄** | 자동 | ★★★ | 추후 도입 |

---

## 1단계. Neon 자동 백업 (기본 제공)

Neon 은 **Point-in-Time Restore(PITR)** 를 기본 제공합니다.
- 일정 기간 내 **특정 시점으로 되돌리기**가 가능합니다. (무료 플랜은 보존 기간이 짧음 — 보통 24시간 내외, 유료는 더 김)
- 추가 설정 없이 자동 적용됩니다.

**복구 방법**
1. Neon 대시보드 → 프로젝트 → **Branches** 또는 **Restore**
2. 되돌릴 시점 선택 → 복원
   - 또는 해당 시점으로 **새 브랜치**를 만들어 데이터를 확인한 뒤 연결 주소를 교체

> ⚠️ 무료 플랜의 보존 기간은 짧으므로, **2단계 수동 덤프를 반드시 병행**하세요.

---

## 2단계. 수동 백업 — `pg_dump` 파일 보관 (권장)

데이터 전체를 **파일 하나(.dump)** 로 내려받아 보관합니다. (엑셀 파일 백업하던 것과 같은 개념)

### 백업 (내보내기)
PostgreSQL 클라이언트가 설치된 PC에서:
```bash
pg_dump "여기에_DIRECT_URL_붙여넣기" -Fc -f daolerp_backup_2026-06-26.dump
```
- `-Fc` : 압축된 커스텀 포맷 (복구가 빠르고 안전)
- 파일명에 **날짜**를 넣어 구분하세요.
- 만들어진 `.dump` 파일을 **회사 공용 드라이브 / 구글드라이브 / 외장하드** 등 2곳 이상에 보관.

> 권장 주기: **주 1회** + 대량 업로드 직후. 최근 4~8개를 돌려가며 보관.

### 복구 (되돌리기)
새(또는 비운) 데이터베이스에 복원:
```bash
pg_restore --clean --if-exists --no-owner -d "여기에_DIRECT_URL_붙여넣기" daolerp_backup_2026-06-26.dump
```

> 복구는 **기존 데이터를 덮어씁니다.** 실수 방지를 위해, 먼저 Neon에서 **새 브랜치/새 DB**에 복원해 확인한 뒤 연결을 바꾸는 방식을 권장합니다.

### CSV 로 가볍게 보관 (선택)
표 단위로 엑셀에서 열어보고 싶다면:
```bash
psql "DIRECT_URL" -c "\copy \"TaxInvoice\" TO 'tax_invoice.csv' CSV HEADER"
psql "DIRECT_URL" -c "\copy \"SalesDeal\"   TO 'sales_deal.csv'   CSV HEADER"
psql "DIRECT_URL" -c "\copy \"BankTransaction\" TO 'bank.csv' CSV HEADER"
psql "DIRECT_URL" -c "\copy \"CardUsage\"   TO 'card.csv'   CSV HEADER"
```

## 3단계. 자동 백업 스케줄 (추후 도입)

규모가 커지면 사람이 매주 떠올려 백업하기 어렵습니다. 자동화 옵션:

- **GitHub Actions 스케줄**: 매일/매주 `pg_dump` 실행 → 결과물을 클라우드 스토리지(R2/S3)에 업로드
- **Neon 유료 플랜**: 보존 기간 연장 + 브랜치 백업 강화
- **외부 백업 서비스**: 관리형 PostgreSQL 백업 도구 연동

> 필요해지면 GitHub Actions 워크플로(`.github/workflows/backup.yml`) 추가를 요청하세요. 자동 구성해 드립니다.

## ✅ 권장 운영 루틴 (요약)

1. **상시**: Neon 자동 백업이 동작 (별도 작업 없음)
2. **주 1회 + 대량 업로드 후**: `pg_dump` 로 `.dump` 파일 생성 → 2곳 이상 보관
3. **분기 1회**: 백업 파일로 **복구가 실제로 되는지 1번 테스트** (가장 중요 — 백업은 복구 검증까지 해야 진짜 백업)
4. 데이터 구조 변경(마이그레이션) 배포 **직전**에도 한 번 백업
